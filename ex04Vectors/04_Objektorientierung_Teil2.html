Ziele¶

Ziel dieser Einheit ist es

  • den Unterschied zwischen Instanz- und Klassenattributen zu verstehen.
  • zu wissen, wie sich Methoden, statische Methoden, und Klassenmethoden unterscheiden.
  • Properties implementieren zu können und deren Nutzen zu verstehen.
  • Vererbung zum Strukturieren von Daten und Funktionalität anwenden zu können.
  • zu verstehen, wie Operatoren für Klassen implementiert werden.

Klassenattribute und -methoden¶

Bisher wurden nur Attribute und Methoden behandelt, die für eine Instanz einer Klasse definiert sind.

Klassenattribute¶

  • Attribute sind auch für Klassen (nicht nur Instanzen) definierbar
  • Klassenattribute sind für alle Instanzen gleich
  • Werden auch als statische Attribute bezeichnet
  • Zugriff über Klasse oder Instanz möglich
In [5]:

Statische Methoden sind weder an eine Klasse noch Instanz gebunden! Können z.B. zur logischen Gliederung von Code verwendet werden.

Klassenmethoden¶

  • Nicht zu verwechseln mit statischen Methoden
  • Nicht an Instanz, aber an Klasse gebunden
  • Erster Parameter (Konvention: cls) ist eine Referenz auf die Klasse
  • Aufruf über Klasse oder Instanz möglich
In [11]:
class Robot:
    __count = 0   # private class/static attribute
    
    def __init__(self):
        type(self).__count += 1
        
    @classmethod
    def robotCount(cls):
        return cls.__count
        
robot1 = Robot()
robot2 = Robot()
print(Robot.robotCount())  # access via class
print(robot1.robotCount()) # access via object

Properties¶

Python-Sprachkonstrukt für einen einfachen, gekapselten Zugriff auf Attribute

  • Anlegen via Python-Funktion property mit zwei Parametern
    • get-Methode für Lesezugriff
    • set-Methode für Schreibzugriff (optional)
  • Ermöglicht Zugriff auf private Attribute so als wären sie public
  • Zugriff erfolgt jedoch gekapselt über get- und set-Methoden

Properties sollten mit privaten get- und set-Methoden kombiniert werden, damit nicht zwei Wege existieren, um auf Attribute zuzugreifen.

In [14]:
class PropertyTest:
    def __init__(self, x):
        self.__x = x      # private attribute x
    
    def __getX(self):     # private get method
        return self.__x
    
    def __setX(self, x):  # private set method
        self.__x = x
    
    x = property(__getX, __setX)  # get and set methods are passed as parameters to property
    
test = PropertyTest(1)
test.x = 5              # access attribute x as if it was public
print(test.x * 4)

        self.setName(name)    # access via set method

        return self.__name

In [16]:
class Robot:
    def __init__(self, name):
        self.name = name  # access via property                 
        
    def sayHello(self):
        print("Hallo, ich bin " + self.name)  # access via property   
        
    def __setName(self, name):  # set method is private
        if isinstance(name, str):
            self.__name = name
        else:
            raise TypeError("name must be a string")
        
    def __getName(self):        # get method is private
        return self.__name
    
    name = property(__getName, __setName)  # property uses get and set method
    
robot = Robot("Marvin")
print(robot.name)                              # access via property  
robot.name = "Marvin, der paranoide Androide"  # access via property  
robot.sayHello()

Attribute: public vs. private mit Kapselung¶

Allgemeines Mantra in den meisten objektorientierten Sprachen

  • Möglichst alle Attribute sollten private sein
  • Zugriff gekapselt über get- und set-Methoden

Möglichkeiten durch Properties in Python

  • Attribute können vorerst als public implementiert werden
  • Sollten Zugriffskontrollen notwendig werden, können die Attribute private gesetzt und via get- und set-Methoden gekapselt werden
  • Die Benutzerschnittstelle ändert sich dabei nicht!

Vererbung¶

Definiert die Beziehung zwischen einer allgemeinen Klasse und einer spezialisierten Klasse

  • Klassen können auf anderen Klassen aufbauen
  • Attribute und Methoden der Basisklasse(n) werden geerbt

Beispiele aus der natürlichen Sprache:

  • Fahrzeug: Auto, Lastwagen, Motorrad, Bus, Zug, Schiff, Flugzeug, ...
  • Flugzeug: Passagierflugzeug, Frachtflugzeug, Kampfflugzeug, ...

Fahrzeug ist die Basis-/Oberklasse, Flugzeug eine Sub-/Unterklasse. Flugzeug kann selbst ebenfalls eine Basisklasse für weitere Subklassen sein.

Beispiel: Basisklasse Konto mit Subklassen Sparkonto und Girokonto

Subklassen erben (übernehmen) Attribute und Methoden der Basisklassen

Überschreiben und Überladen¶

Überschreiben (overwrite)

  • Erlaubt abgeleiteten Klassen, eine eigene Implementierung einer geerbten Methode zu definieren
  • Überschreibende Methode ersetzt dabei die geerbte Methode
  • Geerbte Methode kann in der überschreibenden Methode via super() aufgerufen werden (super() liefert Objekt der Basisklasse)
In [ ]:
class Person:
    def __init__(self, firstName, lastName):
        pass  # do something
    
class Student(Person):
    def __init__(self, firstName, lastName):  # override init method
        super().__init__(firstName, lastName)  # call init method of base class
        pass  # do something more

Beispiel für Überschreiben einer Methode

Überladen (overload)

  • Überladen dient in vielen Sprachen dazu, die gleiche Funktion für verschiedene Parameter zu definieren

Beispiel: C++

int    calc(int a)           { return a + 10; }
double calc(double a)        { return a + 10; }
int    calc(int a, bool sub) { return (sub ? a - 10 : a + 10); }

In Python gibt es kein Überladen von Funktionen und Methoden!

  • Parameter haben keinen deklarierten Typ
  • Unterscheidungen nach übergebenen Parametern kann via Default-Parameter gelöst werden
In [45]:
def calc(a, sub=False):
    if sub:
        return a - 10
    return a + 10    

print(calc(20))
print(calc(20.5))
print(calc(20, True))
print(calc(20, sub=True))

Magische Methoden und Operatorüberladung¶

  • Magische Methoden von Klassen können überschrieben werden
  • Operatoren können für Klassen implementiert werden
  • Operatoren erweitern die Funktionalität von Klassen

Für den Ausdruck $4 + 5 = 9$ ist $+$ der Operator, $4$ und $5$ sind die Operanden.

Bisher behandelte magische Methoden: __init__, __del__, __str__

Vollständige Liste aller magischen Methoden und Operatoren in Python: https://docs.python.org/3/reference/datamodel.html

Einige grundlegende magische Methoden

In [ ]:

object.__init__(self[, ...])     # object initialization
object.__del__(self)             # object deletion
object.__str__(self)             # string representation of object
object.__bool__(self)            # boolean representation of object

class Vector2d:
    def __init__(self, x, y):
        self.x = x
        self.y = y
        
    def __add__(self, other):   # return new object with result
        return Vector2d(self.x + other.x, self.y + other.y)
    
    def __iadd__(self, other):  # modify in-place and return reference to self
        self.x += other.x
        self.y += other.y
        return self

b = Vector2d(10, 20)
c = Vector2d(400, 800)
d = a + b        # __add__ method is called
print(d.x, d.y)

Zusammenfassung¶

  • Klassenattribute sind für alle Instanzen einer Klasse gleich
  • Statische Methoden sind weder an Klasse noch Instanz gebunden
  • Klassenmethoden sind an Klasse gebunden
  • Properties ermöglichen einen einfachen, gekapselten Zugriff auf Attribute
  • Vererbung definiert Beziehung zwischen Basis- und Subklassen
  • Geerbte Attribute und Methoden können überschrieben werden
  • Klassen können von mehreren Basisklassen erben
  • Magische Methoden und Operatoren können Funktionalität einer Klasse erweitern
